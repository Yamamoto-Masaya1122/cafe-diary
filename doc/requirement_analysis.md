
## サービス名 
Cafe Diary 
## サービス概要 
Cafe Diary は、カフェに行く前の情報収集から、訪問後の振り返り・日記記録までを一つのアプリで完結できる、カフェ好きのための記録型カフェサポートサービスです。
Wi-Fiや電源、雰囲気などの条件でカフェを検索でき、訪れた後は写真・感想・評価などを自分だけのカフェ日記として保存できます。
SNSのように公開するのではなく、“自分のカフェ体験を整理・記録するためのプライベートなカフェログアプリ”です。
## 想定されるユーザー層
* カフェ巡りが好きな人
* カフェで作業するのが好きな人 
## サービスコンセプト 
カフェで作業するのが好きでよくカフェに行くのだが、Wi-Fiや電源の有無、雰囲気などの情報を素早く、正確に探すのは難しいと感じていました。また、訪れたカフェをおすすめしようとした際、名前や感想を思い出せず、自分だけの記録の必要性を感じた。本サービスは、カフェを探す前の情報収集から、訪問後の感想・写真・雰囲気などを日記として記録できる“個人専用のカフェログアプリ”。SNSのように他人の目を気にせず、自分の主観で自由に保存できるのが特徴です。カフェ検索では設備・雰囲気・作業向きなど細かい条件から探すことができ、日記を蓄積することで自分専用のカフェ図鑑として活用することができます。 
## 実装を予定している機能 
### MVP 
* 会員登録 
* ログイン（ID、パスワードとGoogleログイン、Xログイン、LINEログインを想定）
* カフェ日記一覧（リスト表示、カレンダー表示） 
* カフェ日記登録 
* カフェ日記編集・削除 
* カフェ日記詳細 
* 店舗一覧・検索 
* お気に入りのカフェ 
* マイページ 
* ユーザー編集 
* 管理画面 店舗登録・編集・削除・一覧 
* 管理画面 カフェ日記登録・編集・削除・一覧 
* 管理画面 ユーザー登録・編集・削除・一覧 
* 管理画面 管理者登録・編集・削除・一覧 

### その後の機能 
* パスワードリセット 
* 経路検索（カフェまでの）

## システム構成

### 使用技術

#### フロントエンド
* TypeScript 5.9.3
* Next.js 15.5.6（App Router想定）
* UIライブラリ：shadcn/ui（Tailwind CSSベース）

### バックエンド
* Next.js API Routes（BFF兼用のバックエンドAPI）

### 認証
* NextAuth.js
（Google、X、LINE、Email / パスワード認証対応）

### データアクセス
* ORM：Prisma 6.19.0
* データベース：PostgreSQL

### インフラ
* Webサーバー / アプリホスティング：Vercel
* DBサーバー：Neon（PostgreSQLマネージドサービス）
* メールサーバー：AWS SES（パスワードリセット時のみ使用）

### ストレージ
* 画像保存：Amazon S3

### 外部API
* 郵便番号から住所取得：ZipCloud（日本郵便公式API）
* ユーザー位置情報取得：HTML5 Geolocation API（ブラウザ標準機能）  
※店舗緯度・経度は管理者が手動入力するため、Google Maps Geocoding APIは使用しない
